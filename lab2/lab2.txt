1. Так как формат FLOAT записывается в виде цепочки из мантиссы и экспоненты, то незначительное изменеие в мантиссе дает весомый вклад в изменение самого числа при большой экспоненте.
Очевидно, погрешность возрастает при сложении чисел существенно разных порядков. Наилучший в смысле точности вариант - сперва складывать числа одного или близких порядков, а потом складывать промежуточные значения.
2. В алгоритме Кэхэна используется отдельная переменная, сохраняющая накопленную на каждом шаге погрешность, которая позволяет восстановить точность на любом шаге вплоть до машинной точности.
3. Я использовал немного другую аналитическую формулу sin((k+1)/2)*sin(k/2)/sin(0.5) (содержащую 2 последовательных умножения), так что ее точность может быть вдвое ниже, тем не менее:

Погрешность прямого суммирования: 2.867999230019597e-11
Погрешность суммирования по возрастанию: 3.4364798059060357e-09
Погрешность суммирования по убыванию: 3.2881793303244358e-09
Погрешность суммирования по Кэхэну: 4.015881508118477e-14

Результаты на базе выборки из 1111111 синусов.

Погрешность увеличилась, так как появилось большое количество элементов выборки разных разрядов, от которого не застрахован даже алгоритм Кэхэна.

4. При сортировке массивов точность алгоритма Кэхэна увеличивается, так как возникновение слагаемых, больших, чем частичная сумма, менее вероятно.

Погрешность суммирования по Кэхэну для массива по возрастанию: 1.2627842702581513e-14
Погрешность суммирования по Кэхэну для массива по убыванию: 1.2627842702581513e-14

5. В первой формуле мы вычитаем числа одного порядка, следовательно, погрешность будет невелика, в то время как рекуррентная формула накапливает существенную погрешность каждый раз, как мы прибавляем число, малое (сильно отличающееся по разряду) по сравнению с накопленным значением дисперсии.
